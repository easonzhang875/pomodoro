## 封面（Cover）

### 1. 图文并茂

**结论：PASS**

- 视觉层（截掉文字后）：蓝色大矩形（左上）、红色矩形（右下）、黄色横条（中段）、蓝色实心圆（左下）、红色三角形（右上）、小正方形线框、底部装饰线。几何色块组合构成完整的包豪斯拼贴画面。
- 文字层（截掉视觉后）：Kicker 标签 "Anthropic · Engineering Deep Dive"、主标题 "提示词缓存决定一切"、副题两行、蓝圆内 "CC" 标识。文字层独立成体系。
- 两者相互叠加而非相互替代，符合"图文并茂"要求。

### 2. 主题忠实

**结论：PASS**

- 全部颜色使用 `var(--ra-color-*)` token，包括 `--ra-color-bg`、`--ra-color-fg`、`--ra-color-accent`、`--ra-color-risk`、`--ra-color-warning`、`--ra-color-border`。
- 字号全部使用 `var(--ra-text-*)` token 或 `clamp()` 表达式（其中 max 值使用 token）。无硬编码像素字号。
- 字重全部使用 `var(--ra-font-weight-bold, 700)`。
- 间距全部使用 `var(--ra-space-*)` token。
- 圆角使用 `var(--ra-radius-md, 0)`。
- 无硬编码字体名（`font-family`）。

备注（非阻塞）：
- 三角形边界的 `clamp(20px, 5vw, 60px)` 和小正方形的 `border: "3px solid ..."` 中出现了绝对像素值。这些属于几何图形维度的必要硬编码（CSS 三角形技巧依赖具体 border 尺寸），不属于"像素字号"禁令范围，且已通过 `clamp()` 提供响应式缩放。但 3px 的 `border` 值若存在对应的 token（如 `--ra-border-width`），建议替换以增强主题切换一致性。

### 3. 内容忠实

**结论：FAIL**

- 文字层忠实传达了文章主题："提示词缓存决定一切" + "六条缓存设计法则" + "反直觉的工程真相"——读者 5 秒内能判断这是一篇关于提示词缓存工程实践的技术长文。
- **但视觉层不通过。** 当前视觉主体由三原色几何色块（蓝矩形、红矩形、黄横条、蓝圆、红三角）组成，属于泛化的包豪斯美学拼贴，与"提示词缓存"这一具体技术主题无概念关联。读者遮住文字只看图形，看到的是一幅抽象几何构成作品，无法联想到"缓存/前缀匹配/命中率/Agent 架构"。
- `cover.md` 对此场景给出了明确参照："文章讲'提示词缓存就是一切' → 封面可以是**缓存命中率曲线 / 重复 token 的高亮带**"。当前实现未采纳此方向。

**必须修复：** 在保留包豪斯几何语法（硬边、三原色、蓝圆、方三角）的前提下，将视觉主体从"泛化抽象几何拼贴"改造为"承载缓存概念的可读图形"。具体建议：
- 将黄色横条改为一段重复的 token 前缀带（连续小方块），从左上延伸到中断点；
- 用红色标记"断裂处"——一个字节标红、之后全部灰色/淡化，呼应前缀匹配原理；
- 蓝色方块和圆可保留为构图元素，但使其充当"缓存命中段"的结构标注；
- 保留三角形和线框小正方形作为点缀（它们承担包豪斯风格定调功能，可以保留）。

### 4. 比例自适应

**结论：PASS**

- 所有内部元素使用百分比定位：`top: "8%"`、`left: "6%"`、`width: "42%"`、`bottom: "14%"`、`right: "5%"` 等。
- 文字层使用 `inset: 0` + flex 布局，不依赖绝对像素。
- 圆形和正方形使用 `aspectRatio: "1"` 保持比例。
- 三角形边界使用 `clamp()` 响应式缩放。
- 底部装饰线使用 `left: "6%"`、`right: "6%"` 相对定位。
- 在 3:4 到 A4/Letter 之间的比例变化中，元素不会溢出错位。

### 5. 不与 Hero 重复

**结论：FAIL**

对比 Cover 与 Hero 的文字内容：

| 位置 | Cover | Hero | 重叠判定 |
|------|-------|------|----------|
| 标题 | "提示词缓存决定一切" | "从零构建 Claude Code 的经验教训：**提示词缓存决定一切**" | Cover 标题是 Hero 标题的后半段子串，直接重复 |
| 副题 | "Claude Code 团队首次公开的**六条缓存设计法则** / 以及那些**反直觉**的工程真相" | "**六条反直觉的** Agent **缓存设计法则** —— 来自 Anthropic 官方的技术深度解读" | 共享"六条""缓存设计法则""反直觉"三个关键短语 |
| Kicker | "Anthropic · Engineering Deep Dive" | Meta 行："Thariq Shihipar · Anthropic" | "Anthropic" 重复出现 |

`cover.md` 明确规定：封面是"钩子"而 Hero 是"锚点"，两者不应做成同一件事。当前 Cover 标题与 Hero 标题共用了完全相同的短语"提示词缓存决定一切"，副题共享了"六条缓存设计法则"+"反直觉"，二者信息高度重叠。读者先看封面、再看 Hero，会感觉读了两次同样的信息。

**必须修复：**
- **Cover 标题方向建议：** 改为更具"钩子"性质的短句，不与 Hero 标题共享整段文字。例如：
  - "缓存活着，产品才活着"——从"生命线"角度切入，与 Hero 标题不重复
  - "一个字节就能毁掉一切"——从"脆弱性"角度切入，制造悬念
  - "10% 的成本，100% 的命脉"——从经济账角度切入
- **Cover 副题方向建议：** 改为更短的 teaser，避免重复 Hero 副题的关键词。例如：
  - "Claude Code 团队从未公开的架构内幕"
  - "为什么你的 Agent 订阅模式跑不通"
- **Cover Kicker：** 将 "Anthropic · Engineering Deep Dive" 改为更短的版本（如仅 "Engineering Deep Dive"），避免与 Hero Meta 中的 "Anthropic" 重复。

  2. **重写封面文字**：Cover 标题改为与 Hero 标题完全不重复的钩子式短句；Cover 副题改为简短的 teaser 而非 Hero 副题的近义改写；Kicker 精简避免与 Hero Meta 中 "Anthropic" 重叠。

  3. Phase 6 Final Review 时务必导出 PDF 目检 Cover 的 `mixBlendMode: "difference"` 在 Chromium print 下的渲染效果。
