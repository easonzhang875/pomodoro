- **主要观点：**
  1. 提示词缓存不是一个"最后加上去"的优化——它是 Agent 产品架构的根本约束，从请求排序到工具设计、从模型选择到上下文管理，缓存考量渗透一切。

- **阅读目标：** 读完能理解 Claude Code 的提示词缓存架构设计逻辑，能将这些模式应用到自己的 Agent 产品中，避免踩同样的坑。
- **版式宽度：** `regular`（~46rem，标准长文阅读宽度。文章以正文叙述为主，表格和代码块适度，regular 足够承载）

- **配图策略：** `none`（技术解释型文章，正文 + Raw 图解 + 表格即可充分表达。架构布局图、数据对比、流程示意全部用 Raw 自由层实现，无需外部图片）

## Theme

- **选定主题：** `bayer`（用户选择。Bayer · 包豪斯 / 三原色几何）
- **理由：** 用户偏好 Bayer 主题的几何构成主义气质——"响亮但有纪律"。"缓存命中率 SEV 告警""3 亿 token 节省"这些硬核工程数据配合三原色几何视觉，会产生一种"工程宣言"的力度。这是一种有意的审美张力：冷静的技术内容 × 热烈的包豪斯表达。
- **与源材料的冲突：** 轻微张力——源文语气克制、工程师笔调，而 Bayer 视觉语言响亮、色彩丰富。处理方式：正文保持克制分析语气，不因主题变浮夸；Raw 和视觉层用 Bayer 的三原色几何语法（蓝圆编号、红蓝黄结构色、硬边方块），让视觉层承载"工程宣言"的气质，文字层保持技术深度。这是"冷静头脑 + 热烈外衣"的混搭。
- **当前信息密度下的表现建议：** 100% longform + bayer → 正文为绝对主体（占 ~80% 篇幅），Raw 在关键概念处点亮节奏（占 ~20%）。Raw 风格：几何流程图、三原色柱状/比例图、蓝圆编号步骤、硬边示意。蓝色做结构和章节圆，红色做风险/破坏警示（缓存断裂、成本爆炸），黄色做色块填充（绝不用于文字）。不使用外部图片。代码块：暖浅 surface + 发丝线，无圆角。禁止斜体（Bayer 协议级禁用）。
